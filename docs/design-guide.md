# jingdu 精讀軟件設計指南

> 2026-07-14 立。回答三個問題：好的精讀軟件需要哪些內容、訓練用戶哪些能力、設計與使用要注意什麼。
> 每次加功能/改流程前對照本文檢查；與本文衝突的改動需先改本文再改碼。

## 1. 訓練目標：七項能力

理論基礎：Nation 四股理論（意義輸入/意義輸出/語言聚焦/流利度各佔約 1/4 時間）、
Krashen 輸入假說（i+1 可理解輸入）、Swain 輸出假說（被迫產出才暴露「懂但不會用」）、
檢索練習效應（測試即學習）、艾賓浩斯間隔重複。

| # | 能力 | 現有環節 | 狀態 |
|---|---|---|---|
| 1 | 聽辨（不看字抓意思） | 聽全文(盲聽)+聽力題 | ✅ |
| 2 | 解碼（詞彙+語法） | 逐句+生詞卡+語法點 | ✅ |
| 3 | 語塊感（collocation） | 連詞成句（部分覆蓋） | ⚠️ |
| 4 | 發音輸出 | 跟讀打分 | ✅ |
| 5 | 記憶鞏固 | 背句+艾賓浩斯錯題本 | ✅ |
| 6 | 主動輸出（說自己的話） | 無 | ❌ 最大缺口 |
| 7 | 遷移（換語境仍會用） | 無（詞只活在本課） | ❌ 缺口 |

## 2. 內容三規則

1. **i+1 難度**：課文生詞率 2–5%；>10% 挫敗機，<2% 學不到。自建課生成時 AI 估難度星級。
2. **舊詞複現 7–12 次**才算習得：新課生成注入舊課詞；復盤混入舊詞新例句。
3. **精讀:泛讀 ≈ 3:7（時間）**：課後 AI 用「僅限學過的詞」生成小故事當泛讀甜點（零版權）。
4. 版權紅線：教材原文一律用戶自供（拍照/粘貼），不由 Claude 批量抓取發佈。

## 3. 一課的學習閉環

```
①預熱 → ②盲聽(建立需求) → ③逐句解碼 → ④操練(生詞/語法/連詞)
→ ⑤輸出(跟讀→背句→造句) → ⑥檢測(聽力題) → ⑦間隔復盤 → ⑧遷移(舊詞新語境)
```

- 聽全文=默認首環節（先盲聽建立需求）——這是有意設計，勿改。
- 環節可跳但有默認順序；低齡跟默認走。

## 4. 使用注意事項

- **節奏**：兒童單次 15–20 分鐘封頂；一課拆 2–3 天正常；頻率 > 單次時長；打卡按「天」記 streak 而非按課。
- **防挫敗**：錯不懲罰只進錯題本；85 分線連續 3 次 80–84 給「很接近了」；語音識別連續失敗 2 次自動出自評按鈕兜底。
- **AI 生成內容必須有校驗層**（2026-07-13 聽力題事故教訓）：程序清洗（下標/格式）+ 聚焦式二次核對（單題單判）雙保險，凡帶標準答案的生成內容缺一不可。
- **聲音是第一介面**：凡 `new SpeechSynthesisUtterance` 必同時設 voice（2026-07-14 教訓）；音頻問題優先級永遠高於視覺問題。
- 雲備份是信任底線：進度丟一次信任就沒了。

## 5. 便捷性檢查清單（每次改 UI 過一遍）

1. 打開即學：首頁第一屏 = 「今天的學習」大按鈕
2. 零打字優先：能點選不打字，能語音不鍵盤（拼寫默寫是刻意練習，除外）
3. 按鈕 ≥44pt，iPad 橫豎屏都測
4. 首頁提示一次「加到主屏幕」（PWA）
5. 離線可學，聯網自動補同步
6. 孩子的每一次選擇都是流失點——下一步永遠只有一個明顯入口

## 7. 朗讀自然度（2026-07-14）

- **根因**：純靜態站無後端，只能用瀏覽器內置 speechSynthesis（離線系統聲），音質有上限，壓縮版最機械。
- **已做**：①選聲/試聽設定頁 voice.html（列設備所有聲音，增強版置頂標「較自然」，試聽+選中存偏好 jingdu_voice_<prefix>，英日各一）②pickVoice 優先序=用戶偏好>高質量關鍵詞(enhanced/siri/neural)>具名優質聲>任一，避開壓縮版③引導下載 iOS/macOS 增強版語音。
- **GLM-TTS 不用於英文**：智譜 2025-12 上線 GLM-TTS（open.bigmodel.cn/api/paas/v4/audio/speech，複用現有 key），但音色全中文（彤彤/jam…），讀英文帶中式口音，不適合英語精讀（小孩學發音口音錯比機械更糟）。若未來要「情感中文旁白」可考慮，英文朗讀靠系統增強版 neural voice 更地道。
- **真人語氣的天花板**：要遠超系統聲，需 neural 雲端 TTS（OpenAI/Edge/ElevenLabs，多語言地道），但都要後端或額外 key，與純靜態+複用單一智譜 key 的架構衝突，暫不做。

- **麥克風教程提示（2026-07-14）**：首次進跟讀環節顯示一次性教程卡（JD.injectMicTip，看過存 jingdu_mic_tip_seen 不再顯示）——說明「第一次要點允許、每次重開會再問是瀏覽器規定非壞掉」+折疊的分設備永久允許方法（iOS啟用聽寫+Safari麥克風允許+加主畫面/Mac Safari網站設定/Chrome🔒）。根因：SpeechRecognition 權限是瀏覽器會話級，網頁無權記住，只能引導用戶在系統設定永久允許。
