# 精讀 jingdu 日語版 — 設計規格（2026-07-04）

## 背景
英語版引擎（新概念二）移植而來，共用 core.js（存儲/艾賓浩斯/雲端同步）與 style.css 基礎樣式，
新增 assets/ruby.js（振假名渲染）與 assets/lesson-jp.js（日語專屬渲染引擎）。

**版權邊界（重要）**：教材《みんなの日本語》等課文內容受版權保護，不可整段抓取/搬運。
每課數據必須由用戶提供內容（拍照/打字某一課），或使用原創內容（不基於任何特定教材）。
本規格文件本身 + jp-01 示例課均為原創，僅用於驗證引擎，不對應教材任何一課。

## 數據 schema（lessons/data/<id>.json）
```json
{
  "id": "jp-01", "num": "第1課", "title": "課名（含中文可選）",
  "sentences": [
    {"jp": "私[わたし]は　学生[がくせい]です。", "romaji": "watashi wa gakusei desu.",
     "zh": "我是學生。", "ana": "講解...",
     "chunks": ["私[わたし]は", "学生[がくせい]です"]}
  ],
  "vocab": [{"w": "私[わたし]", "romaji": "watashi", "pos": "代名詞", "zh": "我", "eg": "..."}],
  "listening": [{"play": [1], "srcIdx": 1, "q": "...", "ans": 0, "opts": ["...","...","...","..."]}],
  "grammar": [{"t": "標題", "body": "<p>...</p>"}]
}
```

- **振假名標記**：`漢字[かな]` → 渲染成 `<ruby>漢字<rt>かな</rt></ruby>`（assets/ruby.js 處理）
- **chunks**：連詞成句用的「文節」切分（2-7 個），不提供則該句不進入連詞成句練習
- **jp 欄位同時扮演三個角色**：①視覺顯示（帶漢字+ruby）②`R.toKana()` 抽取純假名供 TTS/比對 ③`R.toPlain()` 去標記純文字

## 日語專屬技術難點與解法

### 1. 語音（TTS + 識別）
- `JD.speak(text, slow, 'ja-JP')`、`JD.listen(cb, onstate, 'ja-JP')`——core.js 已把 lang 參數化，向後兼容英語版預設 `en-US`
- **⚠️ 已知風險**：iPad Safari 對日語語音識別的準確率未經真實設備驗證，需要用戶實測回饋

### 2. 漢字 vs 讀音比對（本次踩坑修復）
語音識別對日語通常輸出「標準漢字假名混寫」（如「私は学生です」），但比對目標若是純假名讀音
（「わたしはがくせいです」），逐字比對會把完全正確的發音誤判成錯誤。

**解法**：`lesson-jp.js` 啟動時從本課 sentences/vocab 的 `漢字[かな]` 標記自動收集一份
「本課漢字→讀音」對照表（`KANJI_MAP`），辨識結果送進 `JD.compareJP` 前先用這個表把可能出現的
漢字換成讀音再比對。這個對照表也會**跟著錯題一起存進 errbook（`kmap` 欄位）**，
確保之後在復盤中心重測時同樣能正確歸一。

**局限**：僅覆蓋本課出現過的漢字寫法，不是通用日語形態分析（無離線詞典/分詞器）。
換一種漢字寫法、或漢字不在本課範圍內，仍可能誤判——這是為什麼「語音識別環節不準時
自動降級為自評按鈕」的兜底機制格外重要。

### 3. compareJP（core.js 新增）
- `kk2hh()`：片假名→平假名碼位平移（統一片假名/平假名的發音等價）
- 按 Unicode 字符（非空格分詞）做 LCS 逐字比對，返回 ok/miss/bad 三態

### 4. 生詞卡
翻面後看到「中文+詞性+例句」，孩子輸入**平假名讀音**（非拼字母），對比 `kk2hh()` 歸一後的目標讀音。
拼錯 → 存 errbook 時 `en` 欄位存讀音（不是漢字視覺形式），避免復盤時比對到錯誤目標。

### 5. 連詞成句
日語語序特色（動詞在句末）反而讓這個練習更有教學價值。chunks 用「文節」切分而非單詞切分
（如「私は」整塊移動，不拆成「私」+「は」），更符合日語語感。

## 待辦
- 用戶提供《みんなの日本語》具體某一課內容 → 對照本 schema 寫 json → `python3 build_lessons_jp.py <id>`
- iPad 實測日語語音識別準確率，回饋後決定是否需要進一步調整 KANJI_MAP 策略或整體降級為自評
- 若準確率長期不理想，可考慮把「跟讀/背句」的語音識別環節對日語課預設為自評模式（保留識別為可選加分項）
