# 精讀 jingdu — 設計規格（2026-07-04 用戶已確認）

## 目標
給孩子（iPad 為主力設備）用的通用課文精讀站：貼什麼課文精讀什麼。靜態站託管 GitHub Pages（倉庫 `jingdu`，公開），零後端零成本。

## 每課 8 環節（單頁分 tab；2026-07-04 定稿）
0. 聽全文（默認頁）：整篇 TTS 連播+當前句高亮跟燈；盲聽模式（文字模糊）/慢速/循環；完整聽完一遍自動打勾
1. 逐句精讀：點句展開中譯+句型拆解+難點；每句 🔊 TTS 發音（正常/慢速）
2. 生詞卡：音標+詞性+例句，點卡翻面後**必須自評「認識/不認識」**；不認識 → 單詞進錯題本（type:'word'）；全部評完才打勾
3. 語法點：本課 1–3 個，白話講透
4. 口語跟讀：聽 → 跟讀 → 識別逐詞比對，錯詞標紅；**<85% 句子進錯題本**（與背句同 id 自動合併）
5. 聽力題：純聽音答題（不看文本），每題 play:[句索引] 可連播多句；題型=聽句選義+內容理解，4 選 1 即時判分；**答錯 → srcIdx 句進錯題本**
6. 背句挑戰：看句 10 秒（倒計時）→ 蓋住 → 開口背 → 逐詞比對準確率（綠對/紅錯/黃漏）；<85% 進錯題本
7. 完成打卡：七環節進度 + 全清慶祝

## 錯題本兩種類型
- sent（句子）：復盤=看10秒→蓋住→口語背
- word（單詞）：復盤=看中文+詞性→直接開口說英文（到期清單只顯示中文不洩答案）
- 語音識別帶 12 秒安全超時，錯誤訊息分 not-allowed/silence/timeout/aborted 四類白話提示
- 錄音按鈕狀態機（2026-07-04 用戶實測反饋）：按下 → 鎖定 disabled＋「👂 正在聽…」＋結果區顯示「開始讀吧！讀完停一下自動結束打分，不用再按」→ 出分後解鎖變「🎙️ 再試一次」；聆聽中 recBusy 鎖防連按（連按會產生 aborted）

## 系統頁
- index.html：課文目錄 + 每課完成度 + 「📅 今日待復盤 X 句」
- review.html：錯題句按艾賓浩斯節點排隊（30分→1天→2天→4天→7天→15天），到期重背 ≥85% 晉級，6 級全過=牢固

## 技術決策
- 語音識別：webkitSpeechRecognition 單句模式（iPad Safari 需 iOS14.5+ 且開 Siri 與聽寫）；不可用時降級為「自評按鈕」
- TTS：speechSynthesis en-US，rate 0.9/0.6
- 存儲：localStorage（鍵 jingdu_*），單設備；比對：詞級 LCS diff
- 明確不做：推送提醒、跨設備同步、家長遠程報告
- 新課文由 Claude 按本規格生成（後續固化成 jingdu skill），數據結構必須一致

## 加課檢查清單（每課上線前必過）
1. 數據核對：listening 的 play/srcIdx 索引在 sentences 範圍內、ans 正確；vocab 音標/詞性無誤
2. **拼寫歸一實測**：課文若含英式拼寫（-re/-our/-ise/雙寫 l 等）或縮寫，必須逐句過一遍 `JD.compare(en, 美式拼寫版)` 確認 100 分；有新差異詞就加進 core.js norm() 對照表（識別引擎輸出美式）
3. 三處同步：index.html LESSONS 加一行（secs 數）＋（若新增環節）doneCnt key 列表＋lesson.js SEC_LABEL
4. 手機視口實際打開一遍：8 個 tab 全點、背句跑一句、聽力題答一題
